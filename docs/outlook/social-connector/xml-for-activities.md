<span data-ttu-id="ddf78-113">No cenário a seguir, o OSC sincroniza dinamicamente as atividades de uma pessoa selecionada no Painel de Pessoas e obtém detalhes sobre essa pessoa:</span><span class="sxs-lookup"><span data-stu-id="ddf78-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="ddf78-114">Um provedor OSC que oferece suporte à sincronização sob demanda de atividades indica que para o OSC usando os elementos **getActivities** e **dynamicActivitiesLookupEx.**</span><span class="sxs-lookup"><span data-stu-id="ddf78-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="ddf78-115">O provedor OSC também define o elemento **hashFunction**.</span><span class="sxs-lookup"><span data-stu-id="ddf78-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="ddf78-116">Todos os três elementos são elementos filhos de **capabilities**.</span><span class="sxs-lookup"><span data-stu-id="ddf78-116">All three elements are child elements of **capabilities**.</span></span>

3. <span data-ttu-id="ddf78-118">O OSC chama **ISocialProvider::GetCapabilities** para verificar o valor de **getActivities** e **dynamicActivitiesLookupEx** para verificar se o provedor OSC oferece suporte à sincronização sob demanda de atividades.</span><span class="sxs-lookup"><span data-stu-id="ddf78-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="ddf78-119">O OSC também observa o valor do **elemento hashFunction** suportado pelo provedor OSC.</span><span class="sxs-lookup"><span data-stu-id="ddf78-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="ddf78-120">O OSC atualiza o Painel de Pessoas ou o Cartão de Visita para permitir que o usuário veja as atividades mais recentes da pessoa selecionada.</span><span class="sxs-lookup"><span data-stu-id="ddf78-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="ddf78-121">O OSC criptografa o endereço SMTP da pessoa usando a função hash especificada no elemento **hashFunction,** formando uma cadeia de caracteres XML em conformidade com a definição de esquema XML para o elemento **hashedAddresses.**</span><span class="sxs-lookup"><span data-stu-id="ddf78-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="ddf78-122">O OSC chama **ISocialSession2::GetActivitiesEx**, fornecendo essa cadeia de caracteres XML do endereço com hashed Como o  parâmetro _hashedAddresses,_ para obter uma coleção atual de atividades para essa pessoa no parâmetro de atividades.</span><span class="sxs-lookup"><span data-stu-id="ddf78-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="ddf78-123">A _cadeia de_ caracteres de parâmetro de atividades está em conformidade com a definição de esquema XML do elemento **activityFeed.**</span><span class="sxs-lookup"><span data-stu-id="ddf78-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="ddf78-124">Com base na definição de esquema XML para **activityFeed**, o OSC analisará ainda mais a cadeia de caracteres de atividades para descobrir o tipo, a data de publicação e outras informações sobre cada atividade e como exibir a atividade. </span><span class="sxs-lookup"><span data-stu-id="ddf78-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="ddf78-125">Para obter detalhes sobre a pessoa selecionada, o OSC chama [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), fornecendo a mesma cadeia de caracteres XML de endereços com hashed como o argumento para o parâmetro _personsAddresses._</span><span class="sxs-lookup"><span data-stu-id="ddf78-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="ddf78-126">Os detalhes sobre a pessoa são retornados no parâmetro _personsCollection._</span><span class="sxs-lookup"><span data-stu-id="ddf78-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="ddf78-127">Esses detalhes podem incluir **firstName**, **lastName** e **emailAddress**.</span><span class="sxs-lookup"><span data-stu-id="ddf78-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="ddf78-128">O _parâmetro personsCollection_ está em conformidade com a definição de esquema XML para o **elemento person.**</span><span class="sxs-lookup"><span data-stu-id="ddf78-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
