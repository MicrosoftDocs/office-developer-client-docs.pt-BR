<span data-ttu-id="942c8-113">No cenário a seguir, o OSC dinamicamente sincroniza atividades de uma pessoa selecionada no painel de pessoas e obtém detalhes sobre essa pessoa:</span><span class="sxs-lookup"><span data-stu-id="942c8-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="942c8-114">Um provedor OSC que ofereça suporte a sincronização sob demanda das atividades indica que o OSC usando os elementos **getActivities** e **dynamicActivitiesLookupEx** .</span><span class="sxs-lookup"><span data-stu-id="942c8-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="942c8-115">O provedor do OSC também define o elemento **hashFunction** .</span><span class="sxs-lookup"><span data-stu-id="942c8-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="942c8-116">Todos os três elementos são elementos filho de **recursos**.</span><span class="sxs-lookup"><span data-stu-id="942c8-116">All three elements are child elements of **capabilities**.</span></span> 
    
2. <span data-ttu-id="942c8-117">O provedor do OSC implementa os métodos **ISocialProvider::GetCapabilities** e [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) .</span><span class="sxs-lookup"><span data-stu-id="942c8-117">The OSC provider implements the **ISocialProvider::GetCapabilities** and [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) methods.</span></span> 
    
3. <span data-ttu-id="942c8-118">As chamadas OSC **ISocialProvider::GetCapabilities** para verificar o valor de **getActivities** e **dynamicActivitiesLookupEx** para verificar se o provedor OSC suporta a sincronização sob demanda de atividades.</span><span class="sxs-lookup"><span data-stu-id="942c8-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="942c8-119">O OSC também notas o valor do elemento **hashFunction** suportado pelo provedor do OSC.</span><span class="sxs-lookup"><span data-stu-id="942c8-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="942c8-120">O OSC atualiza o painel de pessoas ou o cartão de visita para permitir que o usuário veja as atividades mais recentes da pessoa selecionada.</span><span class="sxs-lookup"><span data-stu-id="942c8-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="942c8-121">O OSC criptografa o endereço da pessoa SMTP usando a função de hash especificada no elemento **hashFunction** , formando uma cadeia de caracteres XML que está em conformidade com a definição de esquema XML para o elemento **hashedAddresses** .</span><span class="sxs-lookup"><span data-stu-id="942c8-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="942c8-122">O OSC chama **ISocialSession2::GetActivitiesEx**, fornecendo essa cadeia de caracteres XML do endereço de hash como o parâmetro _hashedAddresses_ , para obter uma coleção atual das atividades dessa pessoa no parâmetro _atividades_ .</span><span class="sxs-lookup"><span data-stu-id="942c8-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="942c8-123">A cadeia de caracteres do parâmetro de _atividades_ é compatível com a definição de esquema XML do elemento de **feed de atividades** .</span><span class="sxs-lookup"><span data-stu-id="942c8-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="942c8-124">Com base na definição de esquema XML para o **feed de atividades**, ainda mais o OSC analisa a cadeia de caracteres de _atividades_ para descobrir o tipo, publicar data e outras informações sobre cada atividade e como exibir a atividade.</span><span class="sxs-lookup"><span data-stu-id="942c8-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="942c8-125">Para obter detalhes sobre a pessoa selecionada, o OSC chama [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), fornecendo a mesma cadeia XML de hash endereços como o argumento para o parâmetro _personsAddresses_ .</span><span class="sxs-lookup"><span data-stu-id="942c8-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="942c8-126">Os detalhes sobre a pessoa são retornados no parâmetro _personsCollection_ .</span><span class="sxs-lookup"><span data-stu-id="942c8-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="942c8-127">Esses detalhes podem incluir **firstName**, **lastName**e **emailAddress**.</span><span class="sxs-lookup"><span data-stu-id="942c8-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="942c8-128">O parâmetro _personsCollection_ está de acordo com a definição de esquema XML para o elemento de **pessoa** .</span><span class="sxs-lookup"><span data-stu-id="942c8-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
