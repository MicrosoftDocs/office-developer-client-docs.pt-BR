# <a name="pidtagsentrepresentingaddresstype-canonical-property"></a>Propriedade canônica PidTagSentRepresentingAddressType

  
  
**Aplica-se a**: Outlook 2013 | Outlook 2016 
  
Contém o tipo de endereço do usuário de mensagens representado pelo remetente.
  
|||
|:-----|:-----|
|Propriedades associadas:  <br/> |PR_SENT_REPRESENTING_ADDRTYPE, PR_SENT_REPRESENTING_ADDRTYPE_A, PR_SENT_REPRESENTING_ADDRTYPE_W  <br/> |
|Identificador:  <br/> |0x0064  <br/> |
|Tipo de dados:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Área:  <br/> |Endereço  <br/> |
   
## <a name="remarks"></a>Comentários

Essas propriedades são exemplos das propriedades de endereço do usuário de mensagens que está sendo representado pelo remetente. Quando um aplicativo cliente envia uma mensagem em nome de outro cliente, ele deve definir todas as propriedades do remetente representado para os valores desse cliente. Um usuário de mensagens que envia em seu próprio nome normalmente deixa as propriedades do remetente representadas desaperadas.
  
O provedor de transporte de saída deve sempre deixar essa propriedade inalterada se tiver sido definida pelo cliente de envio. Se não for definido, o provedor de transporte deverá defini-lo como a propriedade **PR_SENDER_ADDRTYPE** ([PidTagSenderAddressType](pidtagsenderaddresstype-canonical-property.md)) na cópia de saída da mensagem e deixá-la desa definida na cópia local.
