<span data-ttu-id="1eb08-105">Cada mensagem tem uma propriedade de classe Message, **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), que identifica o tipo, finalidade ou conteúdo da mensagem.</span><span class="sxs-lookup"><span data-stu-id="1eb08-105">Every message has a message class property, **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), which identifies the type, purpose, or content of the message.</span></span> <span data-ttu-id="1eb08-106">**PR_MESSAGE_CLASS** é uma propriedade obrigatória em todas as novas mensagens.</span><span class="sxs-lookup"><span data-stu-id="1eb08-106">**PR_MESSAGE_CLASS** is a required property on all new messages.</span></span> <span data-ttu-id="1eb08-107">Uma classe de mensagem determina o formulário que é usado para apresentar a mensagem ao usuário e a pasta para colocar mensagens de entrada.</span><span class="sxs-lookup"><span data-stu-id="1eb08-107">A message's class determines the form that is used to present the message to the user and the folder for placing incoming messages.</span></span> 
  
<span data-ttu-id="1eb08-108">As classes de mensagem são cadeias de caracteres com distinção entre maiúsculas e minúsculas que contêm caracteres ASCII 32 a 127 e delimitadas por pontos, mas não podem terminar com um ponto.</span><span class="sxs-lookup"><span data-stu-id="1eb08-108">Message classes are case-sensitive character strings that contain ASCII characters 32 through 127 and are delimited by periods, but they cannot end with a period.</span></span> <span data-ttu-id="1eb08-109">Cada cadeia de caracteres representa um nível de subclassificação, e não há limite para o número de níveis permitido.</span><span class="sxs-lookup"><span data-stu-id="1eb08-109">Each string represents a level of subclassing, and there is no limit to the number of levels allowed.</span></span> 
  
<span data-ttu-id="1eb08-110">Por exemplo, a maioria das mensagens que os aplicativos clientes enviam e recebem o outono na classe de mensagem **IPM** , uma categoria ampla que descreve todas as mensagens interpessoais (ou seja, as mensagens que devem ser lidas por um usuário humano, em vez de programaticamente por um computador).</span><span class="sxs-lookup"><span data-stu-id="1eb08-110">For example, most messages that client applications send and receive fall into the **IPM** message class, a broad category that describes all interpersonal messages (that is, messages that are meant to be read by a human user, rather than programmatically by a computer).</span></span> <span data-ttu-id="1eb08-111">Os provedores de repositórios de mensagens descrevem mais precisamente uma mensagem IPM criando uma subclasse **IPM** .</span><span class="sxs-lookup"><span data-stu-id="1eb08-111">Message store providers more precisely describe an IPM message by creating an **IPM** subclass.</span></span> <span data-ttu-id="1eb08-112">A subclasse **IPM** herda as propriedades da classe de mensagem **IPM** .</span><span class="sxs-lookup"><span data-stu-id="1eb08-112">The **IPM** subclass inherits the properties of the **IPM** message class.</span></span> <span data-ttu-id="1eb08-113">As subclasses da classe **IPM** são nomeadas por meio da concatenação de outras cadeias de caracteres no identificador IPM, como **IPM. Observação** para descrever uma mensagem de aviso e **IPM. Contato** para descrever uma mensagem de contato.</span><span class="sxs-lookup"><span data-stu-id="1eb08-113">Subclasses of the **IPM** class are named by concatenating other character strings onto the IPM identifier, such as **IPM.Note** to describe a note message and **IPM.Contact** to describe a contact message.</span></span> 
  
<span data-ttu-id="1eb08-114">Para lidar com a exibição e o gerenciamento de mensagens IPM, os clientes podem usar um formulário padrão que o MAPI fornece.</span><span class="sxs-lookup"><span data-stu-id="1eb08-114">To handle the display and management of IPM messages, clients can use a standard form that MAPI provides.</span></span> <span data-ttu-id="1eb08-115">Para lidar com a exibição e o gerenciamento de novas classes de mensagens, você como desenvolvedor de aplicativo cliente tem duas opções:</span><span class="sxs-lookup"><span data-stu-id="1eb08-115">To handle the display and management of new message classes, you as a client application developer have two options:</span></span>
  
1. <span data-ttu-id="1eb08-116">Você pode criar um novo formulário usando o conjunto de interfaces de formulários definidas por MAPI que um cliente padrão pode usar.</span><span class="sxs-lookup"><span data-stu-id="1eb08-116">You can create a new form by using the set of MAPI-defined form interfaces that a standard client can use.</span></span>
    
2. <span data-ttu-id="1eb08-117">Você pode escrever seu próprio cliente implementando um aplicativo completo e autônomo.</span><span class="sxs-lookup"><span data-stu-id="1eb08-117">You can write your own client by implementing a complete, standalone application.</span></span> 
    
<span data-ttu-id="1eb08-118">Embora os clientes devam definir a propriedade **PR_MESSAGE_CLASS** para cada mensagem de saída para uma subclasse de **IPM** ou **IPC**, o provedor de repositório de mensagens tem a responsabilidade final de defini-la.</span><span class="sxs-lookup"><span data-stu-id="1eb08-118">Although clients should set the **PR_MESSAGE_CLASS** property for every outgoing message to a subclass of either **IPM** or **IPC**, the message store provider has the ultimate responsibility for setting it.</span></span> <span data-ttu-id="1eb08-119">Portanto, se um cliente enviar uma mensagem sem definir sua classe de mensagem, o provedor do repositório de mensagens a definirá como o valor padrão apropriado para o tipo apropriado de cliente.</span><span class="sxs-lookup"><span data-stu-id="1eb08-119">Therefore, if a client sends a message without setting its message class, the message store provider sets it to the appropriate default value for the appropriate type of client.</span></span> <span data-ttu-id="1eb08-120">A classe de mensagem padrão para clientes de mensagens interpessoais é **IPM**; a classe de mensagem padrão para clientes de comunicação entre processos é **IPC**.</span><span class="sxs-lookup"><span data-stu-id="1eb08-120">The default message class for interpersonal messaging clients is **IPM**; the default message class for interprocess communication clients is **IPC**.</span></span> 
  
<span data-ttu-id="1eb08-121">As classes de mensagens têm uma restrição de tamanho de 255 caracteres.</span><span class="sxs-lookup"><span data-stu-id="1eb08-121">Message classes have a length restriction of 255 characters.</span></span> <span data-ttu-id="1eb08-122">No enTanto, as classes de mensagem não devem exceder 127 caracteres para suportar as classes de mensagens usadas nos relatórios.</span><span class="sxs-lookup"><span data-stu-id="1eb08-122">However, message classes should not exceed 127 characters to support the message classes used in reports.</span></span> <span data-ttu-id="1eb08-123">As classes de mensagens de relatório são baseadas na classe da mensagem original, com duas adições: um prefixo e um sufixo.</span><span class="sxs-lookup"><span data-stu-id="1eb08-123">Report message classes are based on the class of the original message, with two additions: a prefix and a suffix.</span></span> <span data-ttu-id="1eb08-124">O prefixo REPORT indica que a mensagem é um relatório e o sufixo indica o tipo de relatório: DR (relatório de entrega), NDR (notificação de não entrega), IPNRN (ler relatório) ou IPNNRN (relatório não lido).</span><span class="sxs-lookup"><span data-stu-id="1eb08-124">The prefix REPORT indicates that the message is a report, and the suffix indicates the type of report: DR (delivery report), NDR (nondelivery report), IPNRN (read report), or IPNNRN (nonread report).</span></span> <span data-ttu-id="1eb08-125">Observe que essas restrições de tamanho são dadas em caracteres; em plataformas que usam um conjunto de caracteres de byte duplo, a contagem de bytes real pode ser maior.</span><span class="sxs-lookup"><span data-stu-id="1eb08-125">Note that these length restrictions are given in characters; on platforms that use a double-byte character set, the actual byte count might be higher.</span></span>
