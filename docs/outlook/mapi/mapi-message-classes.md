<span data-ttu-id="acf74-108">As classes de mensagem são cadeias de caracteres de maiusculas e minúsculas que contêm caracteres ASCII 32 a 127 e são delimitadas por períodos, mas eles não podem terminar com um ponto.</span><span class="sxs-lookup"><span data-stu-id="acf74-108">Message classes are case-sensitive character strings that contain ASCII characters 32 through 127 and are delimited by periods, but they cannot end with a period.</span></span> <span data-ttu-id="acf74-109">Cada cadeia de caracteres representa um nível de subclassificação e não há nenhum limite no número de níveis permitidos.</span><span class="sxs-lookup"><span data-stu-id="acf74-109">Each string represents a level of subclassing, and there is no limit to the number of levels allowed.</span></span>

Observação** para descrever uma mensagem de observação e **IPM. <span data-ttu-id="acf74-114">Para lidar com a exibição e o gerenciamento das mensagens IPM, os clientes podem usar um formulário padrão que fornece de MAPI.</span><span class="sxs-lookup"><span data-stu-id="acf74-114">To handle the display and management of IPM messages, clients can use a standard form that MAPI provides.</span></span> <span data-ttu-id="acf74-115">Para lidar com a exibição e o gerenciamento das novas classes de mensagem, você como um desenvolvedor de aplicativos cliente tem duas opções:</span><span class="sxs-lookup"><span data-stu-id="acf74-115">To handle the display and management of new message classes, you as a client application developer have two options:</span></span>

<span data-ttu-id="acf74-118">Embora clientes devem definir a propriedade **PR_MESSAGE_CLASS** para todas as mensagens de saída para uma subclasse do **IPM** ou **CPI**, o provedor de armazenamento de mensagem tem a responsabilidade ultimate para defini-la.</span><span class="sxs-lookup"><span data-stu-id="acf74-118">Although clients should set the **PR_MESSAGE_CLASS** property for every outgoing message to a subclass of either **IPM** or **IPC**, the message store provider has the ultimate responsibility for setting it.</span></span> <span data-ttu-id="acf74-119">Portanto, se um cliente envia uma mensagem sem a definição de classe da mensagem, o provedor de armazenamento de mensagem define-lo como o valor padrão apropriado para o tipo apropriado de cliente.</span><span class="sxs-lookup"><span data-stu-id="acf74-119">Therefore, if a client sends a message without setting its message class, the message store provider sets it to the appropriate default value for the appropriate type of client.</span></span> <span data-ttu-id="acf74-120">A classe de mensagem padrão para clientes de mensagens interpessoais é **IPM**; a classe de mensagem padrão para clientes de comunicação entre processos é **CPI**.</span><span class="sxs-lookup"><span data-stu-id="acf74-120">The default message class for interpersonal messaging clients is **IPM**; the default message class for interprocess communication clients is **IPC**.</span></span> 
  
<span data-ttu-id="acf74-121">Classes de mensagens tem uma restrição de comprimento de 255 caracteres.</span><span class="sxs-lookup"><span data-stu-id="acf74-121">Message classes have a length restriction of 255 characters.</span></span> <span data-ttu-id="acf74-122">No entanto, as classes de mensagens não devem exceder 127 caracteres para suportar as classes de mensagem usadas em relatórios.</span><span class="sxs-lookup"><span data-stu-id="acf74-122">However, message classes should not exceed 127 characters to support the message classes used in reports.</span></span> <span data-ttu-id="acf74-123">Classes de mensagens de relatório baseiam-se na classe da mensagem original, com duas adições: um prefixo e um sufixo.</span><span class="sxs-lookup"><span data-stu-id="acf74-123">Report message classes are based on the class of the original message, with two additions: a prefix and a suffix.</span></span> <span data-ttu-id="acf74-124">O prefixo relatório indica que a mensagem é um relatório e o sufixo indica o tipo de relatório: recuperação de desastres (relatório de entrega), NDR (relatório de não entrega), IPNRN (leitura relatório) ou IPNNRN (relatório nonread).</span><span class="sxs-lookup"><span data-stu-id="acf74-124">The prefix REPORT indicates that the message is a report, and the suffix indicates the type of report: DR (delivery report), NDR (nondelivery report), IPNRN (read report), or IPNNRN (nonread report).</span></span> <span data-ttu-id="acf74-125">Observe que essas restrições de tamanho são fornecidas em caracteres; em plataformas que usam um conjunto de caracteres de byte duplo, a contagem de bytes real pode ser maior.</span><span class="sxs-lookup"><span data-stu-id="acf74-125">Note that these length restrictions are given in characters; on platforms that use a double-byte character set, the actual byte count might be higher.</span></span>
