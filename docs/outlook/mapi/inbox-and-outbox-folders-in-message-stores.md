<span data-ttu-id="fff94-105">Para ser o repositório de mensagens padrão, um provedor de repositório de mensagens deve implementar as pastas caixa de correio e caixa de saída.</span><span class="sxs-lookup"><span data-stu-id="fff94-105">To be the default message store, a message store provider must implement Inbox and Outbox folders.</span></span> <span data-ttu-id="fff94-106">Normalmente, eles são armazenados na subárvore IPM de um repositório de mensagens.</span><span class="sxs-lookup"><span data-stu-id="fff94-106">They are typically stored in the IPM subtree of a message store.</span></span> <span data-ttu-id="fff94-107">Essas pastas são especiais, pois são designadas como as pastas nas quais as mensagens são entregues e enviadas, mas nenhuma funcionalidade especial é necessária.</span><span class="sxs-lookup"><span data-stu-id="fff94-107">These folders are special in that they are designated as the folders that messages are delivered to and sent from, but no special functionality is required of them.</span></span> <span data-ttu-id="fff94-108">Enviar e receber mensagens ocorre por meio de sequências de chamadas definidas entre aplicativos clientes, o spooler MAPI e o provedor de repositório de mensagens.</span><span class="sxs-lookup"><span data-stu-id="fff94-108">Sending and receiving messages happens by way of defined call sequences between client applications, the MAPI spooler, and the message store provider.</span></span> <span data-ttu-id="fff94-109">As pastas caixa de entrada e caixa de saída são apenas pastas que são usadas para armazenar mensagens durante essas sequências de chamadas.</span><span class="sxs-lookup"><span data-stu-id="fff94-109">The Inbox and Outbox folders are simply folders that are used to hold messages during those call sequences.</span></span> <span data-ttu-id="fff94-110">O ponto importante não é que as pastas sejam especiais ou até que sejam nomeadas como caixa de entrada e caixa de saída; the important point is that the message store provider uses them as part of its support for sending and receiving messages.</span></span>
  
<span data-ttu-id="fff94-111">Para dar suporte ao recebimento de mensagens, o provedor de repositório de mensagens deve implementar os métodos [IMsgStore:: GetReceiveFolderTable](imsgstore-getreceivefoldertable.md) e [IMsgStore:: GetReceiveFolder](imsgstore-getreceivefolder.md) .</span><span class="sxs-lookup"><span data-stu-id="fff94-111">To support receiving messages, the message store provider must implement the [IMsgStore::GetReceiveFolderTable](imsgstore-getreceivefoldertable.md) and [IMsgStore::GetReceiveFolder](imsgstore-getreceivefolder.md) methods.</span></span> <span data-ttu-id="fff94-112">Para obter mais informações, consulte [recebendo mensagens usando provedores de repositório de mensagens](receiving-messages-by-using-message-store-providers.md).</span><span class="sxs-lookup"><span data-stu-id="fff94-112">For more information, see [Receiving Messages by Using Message Store Providers](receiving-messages-by-using-message-store-providers.md).</span></span>
  
<span data-ttu-id="fff94-113">Para dar suporte ao envio de mensagens, o provedor de repositório de mensagens deve suportar o método [IMsgStore:: GetOutgoingQueue](imsgstore-getoutgoingqueue.md) , além tree.</span></span> <span data-ttu-id="fff94-115">No enTanto, é personalizado que um provedor de repositório de mensagens mostre o conteúdo da fila de mensagens de saída na pasta de saída, se houver
