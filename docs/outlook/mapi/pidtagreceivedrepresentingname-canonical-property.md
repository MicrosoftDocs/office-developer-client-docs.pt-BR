# <a name="pidtagreceivedrepresentingname-canonical-property"></a>Propriedade canônica PidTagReceivedRepresentingName

  
  
**Aplica-se a**: Outlook 
  
Contém o nome de exibição para o usuário de mensagens que é representado pelo usuário receptor.
  
|||
|:-----|:-----|
|Propriedades associadas:  <br/> |PR_RCVD_REPRESENTING_NAME, PR_RCVD_REPRESENTING_NAME_A, PR_RCVD_REPRESENTING_NAME_W  <br/> |
|Identificador:  <br/> |0x0044  <br/> |
|Tipo de dados:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Área:  <br/> |Endereço  <br/> |
   
## <a name="remarks"></a>Comentários

Essas propriedades são exemplos das propriedades de endereço para o usuário de mensagens que está sendo representado pelo usuário receptor. Devem ser definidas pelo provedor de transporte de entrada, que também é responsável pela verificação do representante ou de autorização. Se nenhum usuário de mensagens estiver sendo representado, essas propriedades devem ser definidas com o nome de exibição contido na propriedade **PR_RECEIVED_BY_NAME** ([PidTagReceivedByName](pidtagreceivedbyname-canonical-property.md)).
  
Um aplicativo cliente responder a uma mensagem recebida em nome de outro cliente deve copiar essas propriedades de mensagem recebida na propriedade **PR_SENT_REPRESENTING_NAME** ([PidTagSentRepresentingName](pidtagsentrepresentingname-canonical-property.md)) para a resposta.
