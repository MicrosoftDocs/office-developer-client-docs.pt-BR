<span data-ttu-id="dd720-105">Como o seu provedor de armazenamento de mensagem cria e armazena mensagens em que o mecanismo de armazenamento subjacente dependem muito o mecanismo de armazenamento subjacente em si.</span><span class="sxs-lookup"><span data-stu-id="dd720-105">How your message store provider creates and stores messages in the underlying storage mechanism depends heavily on the underlying storage mechanism itself.</span></span> <span data-ttu-id="dd720-106">Em geral, você precisa apenas gravar código para preservar as propriedades de uma mensagem e seus valores.</span><span class="sxs-lookup"><span data-stu-id="dd720-106">In general, you need only to write code to preserve the properties of a message and their values.</span></span>
  
<span data-ttu-id="dd720-107">Quando a mensagem armazenar provedor cria uma nova mensagem, o provedor precisa criar a mensagem com as propriedades necessárias para mensagens.</span><span class="sxs-lookup"><span data-stu-id="dd720-107">When the message store provider creates a new message, the provider needs to create the message with the required properties for messages.</span></span> <span data-ttu-id="dd720-108">Uma lista dessas propriedades pode ser encontrada na documentação para o [IMAPIFolder: IMAPIContainer](imapifolderimapicontainer.md) interface.</span><span class="sxs-lookup"><span data-stu-id="dd720-108">A list of these properties can be found in the documentation for the [IMAPIFolder : IMAPIContainer](imapifolderimapicontainer.md) interface.</span></span> <span data-ttu-id="dd720-109">Depois disso, os aplicativos cliente adicionam as propriedades adicionais com os métodos [IMAPIProp](imapipropiunknown.md) .</span><span class="sxs-lookup"><span data-stu-id="dd720-109">After that, client applications add any additional properties with [IMAPIProp](imapipropiunknown.md) methods.</span></span> 
  
<span data-ttu-id="dd720-110">Quando a mensagem armazena provedor salva uma mensagem para o mecanismo de armazenamento subjacente, o provedor precisa iterar sobre as propriedades da mensagem e salvá-los para o mecanismo de armazenamento subjacente, de forma que eles podem ser totalmente recuperados se a mensagem é aberta mais tarde.</span><span class="sxs-lookup"><span data-stu-id="dd720-110">When the message store provider saves a message to the underlying storage mechanism, the provider needs to iterate over the message's properties, and save them to the underlying storage mechanism such that they can be fully recovered if the message is later opened.</span></span>
  
<span data-ttu-id="dd720-111">MAPI exige que as propriedades nas interfaces [IMessage](imessageimapiprop.md) serão transacionadas, o que significa que as alterações feitas a eles não fiquem permanentes até que o método [IMAPIProp::SaveChanges](imapiprop-savechanges.md) é chamado no objeto de mensagem.</span><span class="sxs-lookup"><span data-stu-id="dd720-111">MAPI requires that the properties on [IMessage](imessageimapiprop.md) interfaces are transacted, meaning that changes made to them do not become permanent until the [IMAPIProp::SaveChanges](imapiprop-savechanges.md) method is called on the message object.</span></span> <span data-ttu-id="dd720-112">O provedor de armazenamento de mensagens é responsável por implementar esse comportamento.</span><span class="sxs-lookup"><span data-stu-id="dd720-112">The message store provider is responsible for implementing this behavior.</span></span> <span data-ttu-id="dd720-113">Geralmente isso não é difícil; Isso simplesmente significa pressionada propriedades na memória, enquanto eles estão sendo modificados e confirmá-los para o mecanismo de armazenamento subjacente quando **SaveChanges** é chamado.</span><span class="sxs-lookup"><span data-stu-id="dd720-113">Usually this is not difficult; it simply means holding properties in memory while they are being modified and committing them to the underlying storage mechanism when **SaveChanges** is called.</span></span> 
  
<span data-ttu-id="dd720-114">Algumas propriedades em objetos de mensagem tem semântica especial para aplicativos de cliente com relação ao método **SaveChanges** , da seguinte maneira:</span><span class="sxs-lookup"><span data-stu-id="dd720-114">Some properties on message objects have special semantics for client applications with respect to the **SaveChanges** method, as follows:</span></span>

- <span data-ttu-id="dd720-117">Algumas propriedades têm relações especiais às propriedades na pasta que estejam em ou para os métodos **IMAPIFolder** .</span><span class="sxs-lookup"><span data-stu-id="dd720-117">Some properties have special relations to properties on the folder they are in or to **IMAPIFolder** methods.</span></span> <span data-ttu-id="dd720-118">Por exemplo, a propriedade **PR_MESSAGE_FLAGS** está relacionada aos sinalizadores usados na chamada [IMAPIFolder::CreateMessage](imapifolder-createmessage.md) .</span><span class="sxs-lookup"><span data-stu-id="dd720-118">For example, the **PR_MESSAGE_FLAGS** property is related to the flags used on the [IMAPIFolder::CreateMessage](imapifolder-createmessage.md) call.</span></span> 
    
- <span data-ttu-id="dd720-119">Algumas propriedades podem não estar disponíveis até que **SaveChanges** é chamado pela primeira vez.</span><span class="sxs-lookup"><span data-stu-id="dd720-119">Some properties may not be available until **SaveChanges** is called for the first time.</span></span> <span data-ttu-id="dd720-120">Por exemplo, a propriedade **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) não estejam disponível até que **SaveChanges** seja chamado.</span><span class="sxs-lookup"><span data-stu-id="dd720-120">For example, the **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property may not be available until **SaveChanges** is called.</span></span> 
    
- <span data-ttu-id="dd720-121">Algumas propriedades podem ter relacionamentos especiais a outras propriedades no objeto de mensagem.</span><span class="sxs-lookup"><span data-stu-id="dd720-121">Some properties can have special relationships to other properties on the message object.</span></span> <span data-ttu-id="dd720-122">Por exemplo, a propriedade **PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)) geralmente é derivada da propriedade **PR_RTF_COMPRESSED** ([PidTagRtfCompressed](pidtagrtfcompressed-canonical-property.md)) em provedores de armazenamento de mensagem que oferecem suporte a mensagens de formato Rich Text.</span><span class="sxs-lookup"><span data-stu-id="dd720-122">For example, the **PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)) property is usually derived from the **PR_RTF_COMPRESSED** ([PidTagRtfCompressed](pidtagrtfcompressed-canonical-property.md)) property in message store providers that support Rich Text Format messages.</span></span>
    
- <span data-ttu-id="dd720-123">Algumas propriedades são usadas por mais de um tipo de objeto relacionado à mensagem repositórios.</span><span class="sxs-lookup"><span data-stu-id="dd720-123">Some properties are used by more than one object type related to message stores.</span></span> <span data-ttu-id="dd720-124">Por exemplo, a propriedade **PR_STORE_SUPPORT_MASK** ([PidTagStoreSupportMask](pidtagstoresupportmask-canonical-property.md)) é necessário em pasta e a mensagem de objetos, bem como a mensagem armazena objetos.</span><span class="sxs-lookup"><span data-stu-id="dd720-124">For example, the **PR_STORE_SUPPORT_MASK** ([PidTagStoreSupportMask](pidtagstoresupportmask-canonical-property.md)) property is required on folder and message objects as well as message store objects.</span></span>
