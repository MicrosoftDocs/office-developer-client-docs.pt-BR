# <a name="pidtagsentrepresentingsmtpaddress-canonical-property"></a>Propriedade canônica PidTagSentRepresentingSmtpAddress

  
  
**Aplica-se a**: Outlook 2013 | Outlook 2016 
  
Contém o endereço de email do protocolo de transporte de email simples (SMTP) para o usuário de mensagens que é representado pelo remetente.
  
|||
|:-----|:-----|
|Propriedades associadas:  <br/> |PR_SENT_REPRESENTING_SMTP_ADDRESS, PR_SENT_REPRESENTING_SMTP_ADDRESS_A, PR_SENT_REPRESENTING_SMTP_ADDRESS_W  <br/> |
|Identificador:  <br/> |0x5D02  <br/> |
|Tipo de dados:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Área:  <br/> |Endereço  <br/> |
   
## <a name="remarks"></a>Comentários

Essa propriedade é um exemplo das propriedades de endereço para o usuário de mensagens está sendo representado pelo remetente. Quando um aplicativo cliente envia uma mensagem em nome de outro cliente, ele deve definir todas as propriedades de remetente representado para os valores para que o cliente. Um usuário de mensagens enviando em seu próprio nome geralmente deixa as propriedades de remetente representado não definidas.
  
O provedor de transporte de saída deve sempre deixar essa propriedade inalterada se ele tiver sido definido pelo cliente de envio. Se ela foi removida, o provedor de transporte deve defini-la como a propriedade **PR_SENDER_SMTP_ADDRESS** ([PidTagSenderSmtpAddress](pidtagsendersmtpaddress-canonical-property.md)) na cópia de saída da mensagem e deixá-lo não definidas na cópia local.
