# <a name="pidtagreceivedrepresentingemailaddress-canonical-property"></a>Propriedade canônica PidTagReceivedRepresentingEmailAddress

  
  
**Aplica-se a**: Outlook 2013 | Outlook 2016 
  
Contém o endereço de email para o usuário de mensagens que é representado pelo usuário receptor.
  
|||
|:-----|:-----|
|Propriedades associadas:  <br/> |PR_RCVD_REPRESENTING_EMAIL_ADDRESS, PR_RCVD_REPRESENTING_EMAIL_ADDRESS_A, PR_RCVD_REPRESENTING_EMAIL_ADDRESS_W  <br/> |
|Identificador:  <br/> |0x0078  <br/> |
|Tipo de dados:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Área:  <br/> |Endereço  <br/> |
   
## <a name="remarks"></a>Comentários

Essas propriedades são exemplos das propriedades de endereço para o usuário de mensagens que está sendo representado pelo usuário receptor. Devem ser definidas pelo provedor de transporte de entrada, que também é responsável pela verificação do representante ou de autorização. Se nenhum usuário de mensagens estiver sendo representado, essas propriedades devem ser definidas como o endereço de email contido na propriedade **PR_RECEIVED_BY_EMAIL_ADDRESS** ([PidTagReceivedByEmailAddress](pidtagreceivedbyemailaddress-canonical-property.md)).
  
Um aplicativo cliente responder a uma mensagem recebida em nome de outro cliente deve copiar essas propriedades da mensagem recebida para a propriedade **PR_SENT_REPRESENTING_EMAIL_ADDRESS** ([PidTagSentRepresentingEmailAddress](pidtagsentrepresentingemailaddress-canonical-property.md)) para o responder.
