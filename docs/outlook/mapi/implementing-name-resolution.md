# <a name="implementing-name-resolution"></a><span data-ttu-id="ff878-103">Implementando a resolução de nomes</span><span class="sxs-lookup"><span data-stu-id="ff878-103">Implementing Name Resolution</span></span>

  
  
<span data-ttu-id="ff878-104">**Aplica-se a**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="ff878-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="ff878-105">Os provedores de agendamento são responsáveis por oferecer suporte à resolução de nomes — o processo de associação de um identificador de entrada a um nome de exibição.</span><span class="sxs-lookup"><span data-stu-id="ff878-105">Address book providers are responsible for supporting name resolution — the process of associating an entry identifier with a display name.</span></span> <span data-ttu-id="ff878-106">Os clientes iniciam a resolução de nomes quando chamam [IAddrBook::ResolveName](iaddrbook-resolvename.md) para garantir que cada membro da lista de destinatários de uma mensagem de saída corresponda a um endereço válido.</span><span class="sxs-lookup"><span data-stu-id="ff878-106">Clients initiate name resolution when they call [IAddrBook::ResolveName](iaddrbook-resolvename.md) to ensure that each member of an outgoing message's recipient list corresponds to a valid address.</span></span> 
  
<span data-ttu-id="ff878-107">Seu provedor pode dar suporte à resolução de nomes:</span><span class="sxs-lookup"><span data-stu-id="ff878-107">Your provider can support name resolution by:</span></span>

<span data-ttu-id="ff878-110">Se você optar por dar suporte a **IABContainer::ResolveNames**, tente localizar uma combinação exata para cada nome de exibição não resolvido na estrutura [ADRLIST](adrlist.md) passada com o parâmetro _lpAdrList._</span><span class="sxs-lookup"><span data-stu-id="ff878-110">If you choose to support **IABContainer::ResolveNames**, attempt to locate an exact match for each unresolved display name in the [ADRLIST](adrlist.md) structure passed in with the  _lpAdrList_ parameter.</span></span> <span data-ttu-id="ff878-111">Você pode identificar um nome de exibição não resolvido porque está faltando a propriedade **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) na matriz de valores de propriedade em seu **membro aEntries** da estrutura **ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="ff878-111">You can identifiy an unresolved display name because it is missing the **PR_ENTRYID** ([PidTagEntryId](pidtagentryid-canonical-property.md)) property in the property value array in its **aEntries** member of the **ADRLIST** structure.</span></span> <span data-ttu-id="ff878-112">Ignore todas as entradas que tenham zero propriedades associadas a elas.</span><span class="sxs-lookup"><span data-stu-id="ff878-112">Ignore any entries that have zero properties associated with them.</span></span> 
  
<span data-ttu-id="ff878-113">Relatar o resultado de sua tentativa de resolução no parâmetro  _lpFlagList,_ uma matriz de sinalizadores que corresponde à matriz de nomes para exibição  _em lpAdrList_.</span><span class="sxs-lookup"><span data-stu-id="ff878-113">Report the result of your attempt at resolution in the  _lpFlagList_ parameter, an array of flags that corresponds to the array of display names in  _lpAdrList_.</span></span> <span data-ttu-id="ff878-114">Os sinalizadores são posicionais de forma que o primeiro sinalizador corresponda ao primeiro membro **aEntries** na estrutura **ADRLIST,** o segundo sinalizador corresponde ao segundo membro **aEntries** e assim por diante.</span><span class="sxs-lookup"><span data-stu-id="ff878-114">The flags are positional such that the first flag corresponds to the first **aEntries** member in the **ADRLIST** structure, the second flag corresponds to the second **aEntries** member, and so on.</span></span> 
  
<span data-ttu-id="ff878-115">Há três resultados possíveis para cada entrada não resolvida:</span><span class="sxs-lookup"><span data-stu-id="ff878-115">There are three possible results for each unresolved entry:</span></span>

- <span data-ttu-id="ff878-118">Várias combinações podem ser encontradas, o que significa que há várias entradas de contêiner que coincidem com a entrada na **estrutura ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="ff878-118">Several matches can be found, meaning that there are multiple container entries that match the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="ff878-119">Defina a entrada correspondente no  _parâmetro lpFlagList_ como MAPI_AMBIGUOUS.</span><span class="sxs-lookup"><span data-stu-id="ff878-119">Set the corresponding entry in the  _lpFlagList_ parameter to MAPI_AMBIGUOUS.</span></span> <span data-ttu-id="ff878-120">Não altere o número de entradas na estrutura **ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="ff878-120">Do not change the number of entries in the **ADRLIST** structure.</span></span> 
    
- <span data-ttu-id="ff878-121">Uma combinação exata pode ser encontrada, o que significa que há apenas uma entrada de contêiner que corresponde à entrada na **estrutura ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="ff878-121">An exact match can be found, meaning that there is only one container entry that matches the entry in the **ADRLIST** structure.</span></span> <span data-ttu-id="ff878-122">Defina o membro correspondente no _parâmetro lpFlagList_ como MAPI_RESOLVED e adicione o identificador de entrada à matriz de propriedades associadas à **entrada ADRLIST.**</span><span class="sxs-lookup"><span data-stu-id="ff878-122">Set the corresponding member in the  _lpFlagList_ parameter to MAPI_RESOLVED and add the entry identifier to the array of properties associated with the **ADRLIST** entry.</span></span> 
    
<span data-ttu-id="ff878-123">Se você optar por não dar suporte a **IABContainer::ResolveNames,** retorne MAPI_E_NO_SUPPORT de sua implementação.</span><span class="sxs-lookup"><span data-stu-id="ff878-123">If you choose not to support **IABContainer::ResolveNames**, return MAPI_E_NO_SUPPORT from your implementation.</span></span>
  
<span data-ttu-id="ff878-124">Todos os provedores de livro de endereços são necessários para dar suporte **à** resolução de nomes ambíguos — PR_ANR restrição de propriedade — nos índices de conteúdo de seus contêineres.</span><span class="sxs-lookup"><span data-stu-id="ff878-124">All address book providers are required to support ambiguous name resolution — the **PR_ANR** property restriction — on their containers' contents tables.</span></span> <span data-ttu-id="ff878-125">Para fornecer esse suporte, manipular a restrição de PR_ANR em sua implementação de [IMAPITable::Restrict](imapitable-restrict.md) executando um tipo de pesquisa de "melhor suposição", correspondendo a uma ou mais propriedades específicas que fazem sentido para seu provedor.</span><span class="sxs-lookup"><span data-stu-id="ff878-125">To provide this support, handle the PR_ANR restriction in your implementation of [IMAPITable::Restrict](imapitable-restrict.md) by performing a "best guess" type of search, matching against one or more particular properties that make sense for your provider.</span></span> <span data-ttu-id="ff878-126">Você pode optar por usar a mesma propriedade ou propriedades sempre, como **PR_DISPLAY_NAME** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) ou **PR_ACCOUNT** ([PidTagAccount](pidtagaccount-canonical-property.md)) ou permitir que um administrador escolha em uma lista de propriedades aceitáveis.</span><span class="sxs-lookup"><span data-stu-id="ff878-126">You can choose to use the same property or properties every time, such as **PR_DISPLAY_NAME** ([PidTagDisplayName](pidtagdisplayname-canonical-property.md)) or **PR_ACCOUNT** ([PidTagAccount](pidtagaccount-canonical-property.md)), or allow an administrator to choose from a list of acceptable properties.</span></span>
