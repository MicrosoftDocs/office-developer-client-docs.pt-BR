# <a name="project-conceptual-and-how-to-articles"></a>Artigos conceituais e de orientação do Project

Encontre links para informações nesta seção sobre as principais partes da plataforma Project 2013, começando com o desenvolvimento CSOM usando JavaScript e o .NET Framework, os complementos do painel de tarefas no Project Professional 2013, a Interface do Project Server e o serviço de relatório OData.
  
O Project Server 2013 se baseia na estrutura do serviço Web da Interface do Project Server (PSI) introduzida no Office Project Server 2007. A plataforma do Project Server 2013 foi criada com o .NET Framework 4. O Project Web App requer o SharePoint Server 2013 como base para a funcionalidade de novos recursos, como o modelo de objeto do lado do cliente (CSOM) para acessar o Project Server na nuvem, versão 4 do Windows Workflow Foundation (WF4) e o serviço OData para as tabelas de banco de dados de relatório. O Project Professional 2013 pode usar os novos Complementos do Office do painel de tarefas.

- A arquitetura e a programação do [Project Server 2013](project-server-2013-architecture-and-programmability.md) descrevem as principais partes da plataforma do Project 2013, incluindo os clientes do Project e o Project Server.
