# <a name="errors"></a>Errors


**Aplica-se a**: Access 2013 | Office 2013

Qualquer operação envolvendo objetos ADO pode gerar um ou mais erros do provedor. Na ocorrência de cada erro, um ou mais objetos **Error** são incluídos na coleção **Errors** do objeto **Connection**. Para obter detalhes sobre como tratar avisos e erros no aplicativo ADO, consulte o [Capítulo 6: Tratamento de erros](chapter-6-error-handling.md).

Um mecanismo individual pode gerar erros de aplicativo. Por exemplo, no Visual Basic, o objeto **Err** conterá os erros no nível do aplicativo.
