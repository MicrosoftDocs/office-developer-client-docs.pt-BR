## <a name="remarks"></a><span data-ttu-id="c1c2d-116">Comentários</span><span class="sxs-lookup"><span data-stu-id="c1c2d-116">Remarks</span></span>

<span data-ttu-id="c1c2d-117">Valores na linha **RelMoveTo** são equivalentes aos valores em uma linha [MoveTo](moveto-row-geometry-section.md) que serão multiplicados pela largura e altura da forma.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-117">Values in the **RelMoveTo** row are equivalent to values in a [MoveTo](moveto-row-geometry-section.md) row that are multiplied by the width and height of the shape.</span></span> <span data-ttu-id="c1c2d-118">Por exemplo: uma linha **RelMoveTo** onde o valor da célula **X** é "0" e o valor da célula **Y** é "0,5" poderia ser substituída pela linha **MoveTo** onde o valor da célula **X** é a fórmula "largura*0" e a célula **Y** é o fórmula "altura*0.5."</span><span class="sxs-lookup"><span data-stu-id="c1c2d-118">For example: a **RelMoveTo** row where the value of the **X** cell is "0" and the value of the **Y** cell is "0.5" could be replaced with **MoveTo** row where the value of the **X** cell is the formula "Width*0" and the **Y** cell is the formula "Height*0.5."</span></span> 
  
<span data-ttu-id="c1c2d-119">A linha **RelMoveTo** contém os *x* e *y* -coordenadas do primeiro vértice da forma se a linha MoveTo for a primeira linha na seção.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-119">The **RelMoveTo** row contains the  *x*  - and  *y*  -coordinates of the first vertex for the shape if the MoveTo row is the first row in the section.</span></span> <span data-ttu-id="c1c2d-120">Normalmente, esse é o primeiro vértice inserido quando a forma foi desenhada e não correspondem necessariamente ao ponto inicial de uma forma 1D.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-120">Usually this is the first vertex placed when the shape was drawn, and it does not necessarily correspond to the begin point of a 1-D shape.</span></span> 
  
<span data-ttu-id="c1c2d-121">Uma seção **Geometry** deve começar com um **MoveTo** ou uma linha **RelMoveTo** , mas você também pode usar a linha de **RelMoveTo** e a linha **MoveTo** para representar um espaçamento na pincelada do caminho de uma forma em relação à largura e a altura da forma.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-121">A **Geometry** section must begin with a **MoveTo** or a **RelMoveTo** row, but you can also use the **RelMoveTo** row and **MoveTo** row to represent a gap in the stroking of a shape's path relative to the shape's width and height.</span></span> <span data-ttu-id="c1c2d-122">No entanto, quando o caminho é usado para definir o limite de uma região preenchida, esse espaçamento é interpretado como um segmento de linha reta.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-122">However, when the path is used to define the boundary of a filled region, this gap is interpreted as a straight line segment.</span></span> <span data-ttu-id="c1c2d-123">Para inserir tal espaçamento, insira uma linha entre duas linhas e altere o tipo de linha para **RelMoveTo**.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-123">To insert such a gap, insert a row between two rows and change the row type to **RelMoveTo**.</span></span> <span data-ttu-id="c1c2d-124">Se a linha **RelMoveTo** for entre duas linhas, que contém o *x* e *y* -coordenadas do primeiro vértice da linha depois da quebra no caminho da forma.</span><span class="sxs-lookup"><span data-stu-id="c1c2d-124">If the **RelMoveTo** row is between two rows, it contains the  *x*  - and  *y*  -coordinates of the first vertex of the line after the break in the shape's path.</span></span>
